# A股智能分析助手

## 快速开始

### 分析股票

用户只需提供股票代码或名称，即可获得完整的多维分析报告：

```
请分析 600519（茅台）
看看珠江股份 600684 怎么样
```

### 验证已有报告

当用户有之前的分析报告时，可以进行数据验证和对比：

```
验证股票研究报告/泰豪科技600590分析报告-20260307.md
对比一下 中钢国际000928 3月7日的报告和今天的数据
```

## Agent Teams 启动判定

本技能采用 Team-First 默认并行流程：

团队触发规则由 `scripts/team_router.py` 统一判定：

- 默认进入 `agent_team`，不再以 `single_flow` 作为主流程分支
- 请求中出现 2 个及以上 A 股代码
- 请求包含“验证/对比/复盘/冲突/分歧/组合/股票池/候选/多股”等关键词
- 请求包含“收集今日数据 + 筛选 + 专家讨论 + 推荐”等复杂串联动作
- 请求满足高意图链路“今日采集 + 10支筛选 + 专家讨论 + 3支推荐”时，强制进入 `full_parallel`
- 高意图链路执行启用连续性守护：禁止回退 `single_flow`、允许节点隔离重试后继续并行汇总

触发后固定技能链路如下：

1. `run_data_auditor`（数据真实性审计前置门禁）
2. `collect_data`
3. `run_fundamental_expert`
4. `run_technical_expert`
5. `run_quant_flow_expert`
6. `run_risk_expert`
7. `run_macro_expert`
8. `run_industry_researcher_expert`
9. `run_event_hunter_expert`
10. `run_expert_identifier_agent`（身份与价格校验，不通过即阻断）
11. `supervisor_review`
12. `render_report`

轻量请求采用同一流程范式的 `lite_parallel` 降级执行，仅减少部分专家节点，不切换为单线流程。

## 短线指标建议与代码入口映射

当用户询问“短线预测还差哪些指标、该改哪里”时，按以下优先级给出建议。

### 必须优先

| 指标 | 数据来源 | 触发规则 | 修改入口 | 输出位置 |
|------|----------|----------|----------|----------|
| VWAP偏离 | 分时价格、成交额、成交量 | \|偏离\|>=2.0%触发强提醒 | `scripts/generate_report.py::_generate_minimal_shortline_recommendation` | 综合结论/短线信号确认 |
| ATR止损 | 近14日高低收 | 止损=当前价-1.5*ATR | `scripts/generate_report.py::_generate_minimal_shortline_recommendation` | 操作建议/止损价 |
| 量比 | 即时成交量、5日均量 | 量比>=1.8且上破压力位 | `scripts/generate_report.py::_generate_minimal_shortline_recommendation` | 技术分析派/触发确认 |

### 建议补充

| 指标 | 数据来源 | 触发规则 | 修改入口 | 输出位置 |
|------|----------|----------|----------|----------|
| 5日资金斜率 | 近5日主力净额 | 斜率由正转负触发降级 | `scripts/generate_report.py::_build_reversal_warning` | 资金流向反转预警 |
| 突破回踩确认 | 支撑位、压力位、分时回落 | 回踩不破且放量恢复 | `scripts/generate_report.py::_build_shortline_signal_lines` | 专家讨论纪要 |

### 可选增强

| 指标 | 数据来源 | 触发规则 | 修改入口 | 输出位置 |
|------|----------|----------|----------|----------|
| 异动公告计数 | 近3日公告与监管提示 | 3日异动公告>=1 | `scripts/generate_report.py::_build_shortline_signal_lines` | 风险提示 |

### 最小改动顺序

1. 先路由：调整 `scripts/team_router.py`，使 team 模式包含短线指标增强任务与主管裁决字段
2. 再指标：在 `scripts/generate_report.py` 增加最小可执行建议函数，产出分层建议与缺失降级
3. 后模板：在 `assets/报告模板.md` 增加短线触发确认与缺失降级说明

### 缺失数据降级规则

- 缺失 `VWAP偏离/ATR止损/量比` 任一关键项时，最终标签上限为 `观察`
- 当 `|VWAP偏离|>=4.0%` 且 `量比<1.0` 时，最终标签上限为 `回避`
- 缺失关键项时，报告必须输出“降级原因”
- 缺失关键项时，置信度上限为 `中`

## 分析流程

### 第一步：获取数据
使用 Web Search 工具获取最新数据（优先可用的搜索工具；若 `mcp__MiniMax__web_search` 不可用，自动切换到当前环境可用的 Web Search）：
- 实时行情、价格涨跌幅
- 资金流向（主力/散户）
- 业绩预告、财务数据（营业收入、同比、环比）
- 行业政策、板块走势
- **重要**：近5日资金流向、近期支撑/压力位、最近一期营收口径

#### 数据提取原则
- 搜索行情时优先包含“股票名称 + 股票代码 + 最新价”关键词组合
- 仅提取“最新价/现价/收盘价/报价”后的价格作为当前价
- 禁止将止损位、历史价、目标价写入当前价格字段
- 若单条结果包含多个股票代码，仅使用当前分析代码对应的数据

### 第二步：专家独立分析
8位专家同时进行独立分析：

| 编号 | 角色 | 背景 | 分析重点 |
|------|------|------|----------|
| 01 | 基本面大师 | 20年价值投资 | 财务数据、估值、行业地位 |
| 02 | 技术分析派 | 15年短线交易 | K线形态、均线、MACD、KDJ |
| 03 | 量化模型师 | 机器学习/统计套利 | 数据驱动、因子分析、资金流向 |
| 04 | 风险控制官 | 18年风控经验 | 下行风险、仓位管理、止损位 |
| 05 | 宏观策略师 | 全球宏观对冲 | 政策面、资金面、市场周期 |
| 06 | 行业研究家 | 产业研究背景 | 行业景气、竞争格局、驱动因子 |
| 07 | 消息面猎手 | 事件驱动交易 | 公告政策、监管信号、事件冲击 |
| 08 | 专家鉴别Agent | 质量门控与一致性校验 | 专家身份、标的一致性、价格锚点偏差 |

#### 预置专家 Agent 调用规则
- 优先从 `c:\Users\Administrator\.claude\agents` 读取专家 Agent
- 推荐映射：
  - `run_data_auditor` → `stock-data-auditor`
  - `run_fundamental_expert` → `stock-fundamental-expert`
  - `run_technical_expert` → `stock-technical-expert`
  - `run_quant_flow_expert` → `stock-quant-flow-expert`
  - `run_risk_expert` → `stock-risk-expert`
  - `run_macro_expert` → `stock-macro-expert`
  - `run_industry_researcher_expert` → `stock-industry-researcher`
  - `run_event_hunter_expert` → `stock-event-hunter`
  - `run_expert_identifier_agent` → `stock-identity-auditor`
- 若某个预置 Agent 缺失，则该角色自动回退为默认内置执行路径，不中断整体 Team 流程
- 输出时应标记角色调用来源（`preconfigured` 或 `default`）便于排障

### 第三步：讨论与整合
- 收集各专家观点
- 识别共识与分歧
- 整合形成最终报告

### 第四步：预测验证（可选）
- 对比历史报告中的关键预测
- 评估准确度
- 总结分析框架的有效性

## 核心分析模块（短线 + 营收双轨）

### ⚡ 时效性与口径警告模块

> 短期技术分析具有时效性，分析时必须包含以下警告：

```
> [!WARNING] 时效性与口径警告
> - 本报告数据截至 {{YYYY年MM月DD日}}
> - 短线信号有效期：**2-3个交易日**
> - 资金流向数据：**每日更新**，需持续跟踪
> - 营收数据有效期：**最新已披露季度**
> - 营收口径：{{单季/累计}}，来源：{{公告/财报/权威平台}}
> - 如需最新数据，请重新获取实时行情
> - 遇到大幅波动（如涨停/跌停）需立即复核
```

### 💰 资金流向反转预警

当检测到资金流向与之前报告不符时，触发预警：

```
> [!WARNING] 资金流向反转预警
> - 报告中主力资金：{{净流入/净流出}}
> - 最新数据显示：{{净流入/净流出}}
> - **方向反转**，需重新评估短期走势
```

### 📊 动态支撑/压力位跟踪

每份报告必须包含动态更新的支撑/压力位：

| 位置 | 价格 | 说明 |
|------|------|------|
| 压力位1 | XX元 | 前期高点 |
| 支撑位1 | XX元 | 前期低点 |

### 🎯 双轨评分与决策模块

统一用双轨评分输出结论，禁止只给主观描述：

| 维度 | 权重 | 得分规则 |
|------|------|----------|
| 短线动量分 | 40% | 近5日主力资金、量价配合、关键位突破 |
| 营收质量分 | 35% | 营收同比/环比、增速变化、预告偏离 |
| 风险约束分 | 25% | 波动率、回撤、公告与监管风险 |

评分输出必须包含：`加权总分` 与 `短线校准后总分`。  
校准规则：弱确认（`|VWAP偏离|>=4.0%` 且 `量比<1.0`）下调 15 分；缺失关键指标下调 5 分；确认充分上调 3 分（上限 100）。

最终标签仅允许：**可做 / 观察 / 回避**，并给出失效条件（如跌破止损位）。

## 🔥 关键预警信号（基于验证经验）

根据2026年3月7-9日的数据验证，发现以下预警信号非常有效：

### 1. 涨停/跌停后的资金流向反转

**预警规则：**
- 涨停当日主力资金：大幅净流入 → 次日需关注是否反转
- 跌停当日主力资金：大幅净流出 → 次日可能反弹

**验证案例：**
- 泰豪科技：3月6日涨停，主力净流入1.67亿 → 3月9日股价回落
- 新华传媒：3月6日涨停，主力净流入1.14亿 → 3月9日跌5.15%

### 2. 压力位触及后的回落

**预警规则：**
- 当股价触及前期压力位（报告预测的止盈位）时
- 需要警惕获利了结导致的回调

**验证案例：**
- 华夏航空：报告预测压力位10.5元 → 3月9日高开触及后回落

### 3. 连续涨停/跌停触发异动公告

**预警规则：**
- 连续3个交易日涨跌幅偏离值累计达20% → 触发异动公告
- 异动公告后需警惕监管风险和获利盘回吐

## 报告验证流程

### 触发条件
当用户要求验证、对比已有报告时，执行以下流程：

### 步骤1：读取历史报告
- 读取用户提供的报告文件
- 提取关键数据：股价、涨跌幅、资金流向、技术判断

### 步骤2：获取最新数据
使用 Web Search 获取今日最新数据：
- 股价和涨跌幅（必选）
- 主力资金流向（必选）
- 近期是否有新公告
- 支撑位/压力位变化

### 步骤3：对比分析
生成对比表格：

| 数据项 | 报告数据 | 最新数据 | 差异 |
|--------|----------|----------|------|
| 股价 | XX元 | XX元 | ±X% |
| 涨跌幅 | X% | X% | 变化 |
| 资金流向 | 主力净流入 | 主力净流出 | 反转 |

### 步骤4：验证结论
- 评估之前预测的准确度
- 识别偏差原因
- 给出更新后的建议

## 跨设备迁移与兼容

为保证在新电脑可直接使用，本技能需遵循以下约束：

- 使用相对路径组织资源，禁止依赖本机绝对路径
- 脚本仅依赖 Python 标准库，不强制第三方依赖
- 数据抓取不绑定单一 MCP 名称，优先使用当前环境可用的 Web Search
- 输出报告目录使用 `stock-reports/` 相对路径
- 若网络受限，允许降级为“仅结构化模板输出 + 明确数据缺失提示”

迁移后建议先做一次冒烟验证：

- 运行 `python -m unittest tests/test_stock_skill.py`
- 用单票与双票 payload 各生成一次报告，确认模板切换和反转预警可用
